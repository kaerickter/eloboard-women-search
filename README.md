# ELOBoard 여성 전적 검색

## 내 컴퓨터에서 실행하기

```powershell
npm start
```

내 컴퓨터에서 열기:

```text
http://localhost:5177
```

같은 와이파이 또는 같은 내부망에 있는 사람은 서버를 켰을 때 표시되는 `LAN:` 주소로 접속할 수 있습니다.
처음 실행할 때 Windows 방화벽 허용 창이 뜨면 허용해야 다른 기기에서 접속할 수 있습니다.

## 컴퓨터를 꺼도 계속 공개하려면

지금 방식은 내 컴퓨터에서 서버를 켜두는 방식입니다.
그래서 컴퓨터가 꺼지거나 인터넷이 끊기면 `localhost` 주소와 `LAN:` 주소는 더 이상 접속되지 않습니다.

계속 공개하려면 이 폴더를 클라우드 서버 또는 VPS에 올려서 24시간 Node.js 서버가 켜져 있게 해야 합니다.

배포할 때 필요한 설정은 아래와 같습니다.

- 실행 명령어: `npm start`
- Node.js 버전: 18 이상
- 포트: 호스팅 업체가 주는 `PORT` 환경변수를 사용하면 됩니다. 이 앱은 이미 `PORT`를 지원합니다.
- 정적 사이트 호스팅만으로는 부족합니다. `/api/data`가 ELOBoard 데이터를 가져와서 처리해야 하므로 Node 서버가 필요합니다.

### 쉬운 공개 방법

1. 이 폴더를 GitHub 저장소에 올립니다.
2. Node.js 앱을 지원하는 호스팅 서비스에 저장소를 연결합니다.
3. 실행 명령어를 `npm start`로 설정합니다.
4. 배포가 끝나면 호스팅 서비스가 제공하는 공개 주소를 다른 사람에게 공유합니다.

### VPS로 공개하는 방법

1. 저렴한 VPS를 하나 준비합니다.
2. 서버에 Node.js 18 이상을 설치합니다.
3. 이 폴더를 서버에 올립니다.
4. `npm start`로 실행합니다.
5. 계속 켜두려면 `pm2` 같은 프로세스 관리 도구를 사용합니다.
6. 도메인을 연결하면 더 보기 좋은 주소로 공유할 수 있습니다.

## 주요 기능

- ELOBoard 여성 전적 데이터에서 선수 이름을 검색합니다.
- 선수 프로필 페이지의 `wr_id`를 찾아 전적을 불러옵니다.
- 연도별/월별 총전적, 승, 패, 승률을 보여줍니다.
- 연도/월 드롭박스를 바꾸면 선택 기간 전적 목록도 함께 바뀝니다.
- 상대 이름을 입력하면 최근 90일 상대전적의 승, 패, 승률을 보여줍니다.
- 상대전적 화면의 `남성전적` 탭에서 선수 1·2와 프로리그 조건으로 남성 전적을 검색합니다.
- `/api/men/options`와 `/api/men/records`가 eloboard 남성 공개 전적을 조회해 종족별 및 상대 선수별 결과를 제공합니다.
- 원본 선수 프로필을 열 수 있습니다.
- `매치 점수판` 메뉴에서 참가자 수와 게임 수를 조절할 수 있습니다.
- 게임별 점수와 코멘트, 자동 합계가 같은 브라우저에 저장됩니다.
- 게임 종료 후 번호를 랜덤 배정하고 홀수팀·짝수팀 점수를 합산해 승리팀을 표시합니다.
- 번호 추첨 후 참가자와 입력 내용을 1, 2, 3, 4 순서로 함께 정렬할 수 있습니다.

점수판 주소:

```text
/scoreboard.html
```

## 대학대결

`/university-matchup.html`에서 FA를 제외한 현재 대학 두 곳을 선택하면 동일 티어 선수끼리의 모든 교차 조합을 조회합니다.

- 대학과 선수 명단은 `eloboard.com/univ`의 공개 스폰현황에서 검색할 때 불러옵니다.
- 이름 끝 괄호를 티어로 사용합니다. 예: `려원님(6)` → 선수 `려원님`, 티어 `6`.
- 숫자 티어는 여성·혼성 상대전적, 문자 티어(갓/킹/잭/조커)는 남성 상대전적에서 조회합니다.
- 각 선수 조합의 총 전적과 최근 90일 전적, 티어별 합계, 대학 전체 합계를 표시합니다.
- 같은 티어가 A대학 2명, B대학 3명이면 2 × 3 = 6개 조합을 모두 표시합니다.

### 로컬 실행

```powershell
npm start
```

브라우저에서 `http://localhost:5177/university-matchup.html`을 엽니다. 정적 HTML만 직접 열면 외부 데이터를 중계하는 API를 사용할 수 없으므로 반드시 Node 서버로 실행해야 합니다.

### Render 배포

기존 Render 서비스에 이 폴더를 반영한 뒤 다시 배포합니다. Build Command는 비워 두거나 `npm install`, Start Command는 `npm start`로 설정합니다. 서버는 Render가 제공하는 `PORT` 환경 변수를 자동으로 사용합니다. 별도의 데이터베이스나 비밀키는 필요하지 않습니다.

추가된 API:

- `GET /api/universities`: FA 제외 대학 목록
- `GET /api/universities/roster?name=JSA`: 대학별 선수·티어 목록
- `POST /api/universities/matchup`: 두 대학의 전체 동일 티어 전적 집계
